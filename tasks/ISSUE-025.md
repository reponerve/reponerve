# ISSUE-025 — Context Templates

## Objective

Create deterministic rendering templates for Repository Context.

This layer is responsible for converting:

```go
RepositoryContext
```

into presentation-ready output.

---

# Background

The Context Generator produces structured data.

The Template Layer produces human-readable output.

Separating generation from rendering enables:

* CLI output
* File export
* MCP responses
* Agent integrations

to reuse the same RepositoryContext model.

---

# Input

```go
RepositoryContext
```

Generated by:

```text
internal/context/generator
```

---

# Output

Markdown.

Example:

# Repository Context

Repository: repo_xxx

Generated: 2026-06-06T12:00:00Z

## Key Decisions

* Use Redis Cache
* Adopt gRPC

## Key Intents

* Reduce Latency
* Improve Reliability

## Key Facts

* Auth Service USES Redis
* API Gateway DEPENDS_ON Auth Service

## Recent Events

* Introduce Redis Cache
* Refactor Authentication Flow

---

# Renderer

Create:

```go
type Renderer struct {
}
```

Responsibilities:

* Accept RepositoryContext
* Produce deterministic markdown output
* Preserve ordering from the generator

---

# Package Structure

Recommended:

```text
internal/context/render/

renderer.go
```

or equivalent structure.

Render as bullet lists.

## Events

Render as bullet lists.

# Constraints

* HTML
* JSON
* AI summarization
* Natural language generation
* MCP formatting

Markdown only.
