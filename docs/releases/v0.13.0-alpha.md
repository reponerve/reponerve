# Release v0.13.0-alpha — Token Intelligence (ISSUE-060)

Date: 2026-06-11

Milestone: ISSUE-060 Token Intelligence Layer

---

## Summary

Closes ISSUE-060 acceptance criteria: graph-aware context compression, MCP/CLI format parity, compact output ≥50% reduction, post-commit hooks, and incremental scan via `reponerve scan`.

---

## Features

### Output formats (CLI + MCP)

- `--format prose|json|compact` and `--token-budget N` on all Development Experience CLI commands
- MCP DE tools accept `format` and `token_budget` on `ask`, `explain`, `plan`, `review`, `onboard`, and related tools
- `formatted` field in MCP envelope respects compact + budget

### Graph-aware compression

- `internal/agent/compression/` ranks decisions, facts, intents, and events by topic relevance
- Relationship graph boosts scores for linked memories
- `generate_context` MCP accepts `topic` and `token_budget` for compressed context packs

### Compact format

- Collapses evidence lines, caps related-entity lists, shortens headers and paths
- ≥50% size reduction vs prose on realistic DE output (tested)

### Agent hooks

- `reponerve hook install|uninstall|status` — post-commit `reponerve scan`
- Documented for Cursor and Claude Code in `docs/ai-chat-integration.md`

### Ask routing fix

- `Why do we use SQLite?` routes to `decision_rationale` (use/useing verb forms)

## Upgrade

```bash
go install github.com/reponerve/reponerve/cmd/reponerve@latest
reponerve hook install   # optional: refresh memory after each commit
```

Token-efficient MCP example:

```json
{ "name": "ask", "arguments": { "question": "Why do we use SQLite?", "format": "compact", "token_budget": 1500 } }
```
