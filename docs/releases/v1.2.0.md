# RepoNerve v1.2.0

Release Date: 2026-06-24

Status: **Released**

---

# Summary

v1.2.0 completes **Native Development Discipline** (RFC-003 Phase B/C). Agents get structured **Reuse Protocol** and **Ship Readiness** packs from repository evidence — not generic “search first” or “review before merge” prompts.

---

# Highlights

## Reuse Protocol (RFC-003 Phase B)

* CLI: `reponerve reuse-check "<intent>" --json`
* MCP: `reuse_check` with `intent`
* Ranked `reuse_candidates` with `defined_in`, `role`, and evidence
* Agent kind `reuse_check` with `must_use_before_edit` when candidates exist

## Ship Readiness (RFC-003 Phase C)

* CLI: `reponerve ship-check "<topic>" --json`
* MCP: `ship_check` with `topic`
* Structured `ship_blockers` and `advisories` from review + impact orchestration
* Migration/schema ADR detection, blast-radius advisories, ownership and rollback reminders

## Integration updates

* Discipline rules and skills reference `reuse-check` / `ship-check` (no longer planned-only)
* **47 MCP tools** total (+2)
* Structured pruning caps for reuse and ship-check payloads

---

# Upgrade notes

* Re-run `reponerve integrate --force` in existing projects to refresh discipline rules and skill files
* MCP consumers: handle new `reuse_check` and `ship_check` tools and envelope kinds
