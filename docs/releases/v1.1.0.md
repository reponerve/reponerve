# RepoNerve v1.1.0

Release Date: 2026-06-24

Status: **Released**

---

# Summary

v1.1.0 makes RepoNerve more **agent-consumable** and ships **Native Development Discipline** by default. Development Experience responses are bounded, feature topics resolve to evidence-backed feature packs, and `reponerve init` installs discipline rules so every repository gets plan/review habits without extra setup.

---

# Highlights

## Bounded agent responses (RFC-001)

* Default **token budget 1500** when unset (CLI + MCP)
* Structured JSON pruning with caps on `related`, `evidence`, and plan fields
* `agent.truncated`, `agent.prefer_narrow_tools`, and `completeness: partial` when capped
* Formatted output caps for long entity and plan sections

## Feature Intelligence v2 (RFC-002)

* Derive features from expertise domains, `FEATURE_INTRODUCED` events, and decision titles
* CLI: `reponerve list-features`, `reponerve explain-feature`
* MCP: `list_features`, `explain_feature` (**45 tools** total)
* `explain [topic]` auto-routes to features only when `ShouldAutoExplain` passes (exact name or single-token alias)

## Native Development Discipline — Phase A (RFC-003)

* `reponerve init` bundles `.cursor/rules/coding-guidelines.mdc` and `development-discipline.mdc`
* Skill + reference document plan / review / impact workflow
* **Reuse Protocol** and **Ship Readiness** habits via existing DE tools (structured `reuse-check` / `ship-check` planned for v1.2)

## Documentation

* RFCs 001–003
* Post-1.0 semver policy: `docs/releases/versioning.md`
* Release status docs updated (`v1.0.0` tagged; latest line through v1.1.0)

---

# Upgrade notes

* MCP tool count increases from 43 to 45 (`list_features`, `explain_feature`)
* DE tools may return smaller `structured` payloads by default; check `agent.truncated` and use narrow tools (`explain_function`, `explain_file`) when directed
* Re-run `reponerve init` or `reponerve integrate --force` in existing projects to install discipline rules
