# Project Overview

RepoNerve is a software understanding platform built around knowledge preservation.

RepoNerve preserves, organizes, and transfers software knowledge so that understanding survives beyond individual contributors. It serves as a **software memory system** — transforming repository artifacts and source code into durable intelligence, then delivering Software Understanding through Development Experience.

```text
Knowledge Preservation → Software Memory → Software Understanding
```

---

# Product Mission

Software Understanding is the primary outcome. Repository Intelligence is one completed capability. ISSUE-057 delivers Code Intelligence, Repository-Code Linking, Feature Understanding, and Development Experience required to complete v1.0.

# Architecture Summary

```text
Repository
    ↓
Ingestion
    ↓
Memory + Code Indexing
    ↓
Repository Intelligence + Code Intelligence
    ↓
Repository-Code Linking
    ↓
Feature Understanding (via Development Experience)
    ↓
Development Experience
    ↓
MCP / CLI
    ↓
AI Agents
```

Repository Intelligence and Code Intelligence remain authoritative sources.

Development Experience is the orchestration layer.

# Known Limitations

* Code Intelligence is not yet implemented.
* Repository-Code Linking is not yet implemented.
* Feature Understanding is not yet implemented.
* Development Experience CLI commands are not yet implemented.
* Search remains deterministic and evidence-based only.
* Semantic search, hybrid search, and search adapters are intentionally excluded from v1.0.0.
* Sessions and workflows are in-memory packaging artifacts and are not persisted.
* RepoNerve remains local-first and does not introduce hosted infrastructure.
