# RepoNerve v0.12.0-alpha

Release Date: 2026-06-11

Status: Released (engineering iteration — ISSUE-057 v0.12 scope)

Milestone: Development Experience + Repository-Code Linking

---

## Summary

v0.12.0-alpha delivers the Development Experience product surface and exposes it to AI agents through MCP. Users and agents can ask questions, explain code, plan changes, prepare reviews, and analyze topic impact using evidence from Code Intelligence and Repository Intelligence.

This alpha validates ISSUE-057 acceptance criteria. It is not the v1.0.0 product release.

---

## Features

### Development Experience (CLI)

- `ask` — evidence-backed Q&A across repository memory and code
- `explain` — topic explanations with code and memory context
- `explain-file`, `explain-function`, `explain-struct`, `explain-interface`, `explain-type` — symbol- and file-scoped explanations
- `plan` — development plans grounded in repository structure
- `review` — review guides with linked decisions, code, and ownership
- `impact` — topic-based impact analysis (distinct from graph `analyze_impact`)

- `onboard` — day-one repository orientation for humans and agents

### Development Experience (MCP)

Eleven MCP tools mirror the CLI commands:

- `ask`, `explain`, `explain_file`, `explain_function`, `explain_struct`, `explain_interface`, `explain_type`, `plan`, `review`, `analyze_topic_impact`, `onboard`

MCP server registers **38 tools** total. Shared wiring lives in `internal/cli/devwire/wire_service.go`.

### Repository-Code Linking

- Deterministic links between ADRs, decisions, facts, events, and code entities
- Reduced link noise for commit events in plan/review output

### Code Intelligence polish

- Ambiguous symbol disambiguation with qualified-name hints and `--package` filter
- Human-readable entity labels (paths over hash IDs)
- Method call resolution improvements in call graph indexing
- Transactional code index replacement (atomic reindex)

### Production hardening

- Ownership/expertise recomputed from full persisted memory on every scan (incremental-safe)
- `FlexibleTime` on all SQLite reader datetime fields
- FTS search scoped per repository; ADR body indexed for rationale queries
- SQLite `busy_timeout` and single-writer connection pool
- CI test workflow (`.github/workflows/test.yml`)

### Agent integration

- `.cursor/skills/reponerve/SKILL.md` — context-first workflow for Cursor agents
- Agent context contract (`docs/architecture/agent-context-contract.md`)

### ADR scaffolding

- `docs/adr/` with four architecture decision records for RepoNerve itself
- Populated by `reponerve scan` to enable rationale in `ask` and search

### Testing

- Integration acceptance tests: `tests/integration/de_acceptance_test.go` (semantic: ADR rationale, rescan, homonyms, task intake)
- Fixture repository: `tests/fixtures/de-acceptance/`
- MCP Development Experience handler tests
- `go test ./...` and `go test -race ./...` pass

## Upgrade Notes

1. Rebuild and install the CLI.
2. Run `reponerve scan` to ingest `docs/adr/` and refresh code indexes.
3. Restart the MCP server in VS Code after upgrading.
4. Copilot Chat agents can use Development Experience tools without shelling out to CLI commands.
