# RepoNerve v1.3.0

Release Date: 2026-06-24

Status: **Planned**

---

# Summary

v1.3.0 completes **Native Development Discipline** repo-adaptive policy (RFC-003 Phase D), **Team Delivery Intelligence** (RFC-004), and **Configurable Document Paths** (RFC-005) for ADR/decision ingestion on real-world repo layouts.

---

# Highlights

## Repo-adaptive discipline policy (RFC-003 Phase D)

* `scan` writes `.reponerve/discipline-policy.json` (ADR dir, CI workflows, layer conventions, ship hints)
* DE envelopes include `agent.discipline_policy`
* CLI: `reponerve discipline-policy --json`

## Evidence Review (RFC-004 Phase A)

* `review` / `reponerve review` returns `discipline_checks` from repository policy
* `recommended_next_tools`: `ship_check`, `reuse_check`, `analyze_topic_impact`
* Agent guidance to apply discipline checks before merge

## PR Context (RFC-004 Phase B)

* CLI: `reponerve pr-context [--file path ...] [topic] --json`
* MCP: `pr_context` with `changed_files` and optional `topic`
* Structured pack: review + ship_check + `pr_comment_markdown` for CI

## GitHub Action template (RFC-004 Phase C)

* `reponerve integrate` installs `.github/workflows/reponerve-pr.yml.example`
* Documented flow: scan → pr-context → PR comment

## Integration updates

* **48 MCP tools** total (+1 `pr_context`)
* Skill and discipline rules reference `pr-context` for PR / merge workflows

## Configurable document paths (RFC-005)

* `ingestion.document_paths` in `.reponerve/config.yaml` — merge custom ADR/architecture directories with defaults
* Expanded defaults: `docs/decisions`, `docs/rfc` (indexes this repo’s RFCs without config)
* Discipline policy and scan share the same resolved path list

```yaml
ingestion:
  document_paths:
    - path: docs/decisions
      kind: adr
```
