# Agent Intelligence V1

## Purpose

Enable AI coding agents to reason over repository knowledge generated by RepoNerve.

The Agent Intelligence layer sits above:

* Memory Engine
* Query Engine
* Context Engine
* MCP Server

and provides higher-level repository understanding.

---

# Architecture

Repository
↓
Memory Engine
↓
Query Engine
↓
Context Engine
↓
MCP Server
↓
Agent Intelligence
↓
AI Coding Agents

---

# Goals

Provide agent-oriented capabilities:

* Repository onboarding
* Repository Q&A
* Change impact analysis
* Architectural guidance
* Context compression

without introducing model training or custom LLMs.

---

# Design Principles

## Reuse Existing Knowledge

Agent Intelligence must consume:

* Memory Engine
* Query Engine
* Context Engine

It must not duplicate extraction logic.

---

## Deterministic First

Prefer deterministic reasoning before introducing AI summarization.

Examples:

* Relationship traversal
* Dependency tracing
* Context assembly
* Decision explanation

---

## LLM Optional

V1 should work without any external LLM.

LLMs may enhance responses but should not be required.

---

# Capabilities

## Repository Onboarding

Answer:

* What is this repository?
* What are the important decisions?
* What are the important facts?

---

## Repository Q&A

Answer questions using:

* Memory
* Relationships
* Context

---

## Impact Analysis

Determine:

If X changes:

* Which decisions are affected?
* Which facts are affected?
* Which events are related?

---

## Architectural Guidance

Explain:

* Why a decision exists
* What supports a decision
* Which intents drove a decision

---

## Context Compression

Generate smaller agent-focused context packages.

Useful for:

* Token limits
* Long repositories
* MCP consumers

* Custom LLM training
* Embeddings

* Ownership extraction
