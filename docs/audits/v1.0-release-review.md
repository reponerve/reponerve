# Token Economics

Implementation levers documented in `docs/product/token-economics.md` are shipped:

* Deterministic extraction (no LLM scan)
* Pre-indexed repository memory
* Token-budget context packs and compact format
* Incremental scan via `reponerve hook install`

Quantitative production benchmarks (80–95% exploration token reduction) remain design targets; compression behavior is validated in unit tests.

---

# Known Limitations (Agreed v1.0 Exclusions)

* No semantic / hybrid embedding search
* No user-defined workflow composition (fixed templates only)
* No autonomous code modification or deployment
* No cloud-required core product
* No cross-repo enterprise federation
* Bash/shell scripts indexed for functions only (no full shell semantics)
* SQL indexing covers DDL objects (tables, views), not query plans
