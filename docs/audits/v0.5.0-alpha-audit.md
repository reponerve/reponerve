# Package Boundaries

## Assessment

Status: Healthy

Current layering:

cmd/
↓
internal/
↓
pkg/models

Observed strengths:

* CLI commands remain thin.
* Business logic lives outside command handlers.
* Context Engine is separated from Query Engine.
* MCP layer consumes existing engines rather than duplicating logic.
* Memory extraction remains isolated from storage.

Example:

Context CLI
↓
Context Generator
↓
Context Reader
↓
Query Readers

This follows intended architecture.

## Findings

No major package boundary violations identified.

## Recommendation

Continue enforcing:

Storage
↓
Memory
↓
Query
↓
Context
↓
MCP

Never allow reverse dependencies.

# SQLite Coupling

## Assessment

Status: Good

Observed improvements:

* Discovery decoupled from SQLite.
* Scanners decoupled from SQLite.
* Context Engine uses readers.
* MCP layer avoids direct SQL access.

## Remaining Coupling

SQLite remains the only storage implementation.

This is acceptable for v0.5.0-alpha.

## Recommendation

Do not abstract storage further yet.

Only introduce alternative storage implementations when a real requirement exists.
