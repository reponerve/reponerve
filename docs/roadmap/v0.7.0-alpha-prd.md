# RepoNerve v0.7.0-alpha PRD

## Objective

Introduce Ownership Intelligence to the repository knowledge graph.

RepoNerve currently answers:

* What happened?
* Why did it happen?
* What is affected?

Ownership Intelligence enables RepoNerve to answer:

* Who contributed?
* Who has expertise?
* Who should review changes?
* Who is most familiar with a repository area?

---

# Problem

Current repository intelligence lacks the human dimension.

RepoNerve understands:

* Events
* Decisions
* Intents
* Facts
* Relationships

but cannot associate repository knowledge with contributors.

As a result:

* Expertise is hidden
* Ownership is implicit
* Review recommendations are impossible
* Repository knowledge lacks contributor attribution

---

# Solution

Build Ownership Intelligence.

Ownership Intelligence derives contributor and expertise information from repository evidence and integrates it into the existing memory graph.

Ownership must be:

* Deterministic
* Evidence-based
* Traceable
* Explainable

---

# Scope

## ISSUE-037 — Contributor Model

Create ownership domain models.

Introduce:

* Contributor
* Expertise

Add required storage interfaces and migrations.

---

## ISSUE-038 — Contributor Extraction

Extract contributors from repository artifacts.

Initial sources:

* Git commit history

Generate contributor records.

Compute:

* First Seen
* Last Seen
* Commit Count

---

## ISSUE-039 — Expertise Detection

Detect contributor expertise.

Generate:

* Knowledge Domains
* Expertise Scores

Using:

* Contribution frequency
* Repository evidence
* Historical activity

No AI or LLM scoring.

---

## ISSUE-040 — Ownership Query Engine

Expose ownership intelligence through query APIs.

Examples:

* List contributors
* Get contributor
* List expertise
* Contributor trace

---

## ISSUE-041 — Ownership MCP Tools

Expose ownership intelligence through MCP.

Examples:

* list_contributors
* get_contributor
* list_expertise
* recommend_reviewers

---

# User Stories

## Repository Maintainer

As a maintainer,

I want to understand who contributes to specific repository areas,

so that I can make informed review decisions.

---

## New Developer

As a new contributor,

I want to know who understands a repository area,

so that I can ask the right questions.

---

## AI Coding Agent

As an AI agent,

I want to identify knowledgeable contributors,

so that I can recommend reviewers and provide better guidance.

---

# Out Of Scope

The following are not included in v0.7.0-alpha:

* AI-generated ownership
* Embeddings
* Vector search
* Team hierarchy modeling
* Organization charts
* Automatic code ownership enforcement

---

Who contributed?

Who authored this decision?

Who has expertise in this domain?

Who should review this change?

using repository evidence.
