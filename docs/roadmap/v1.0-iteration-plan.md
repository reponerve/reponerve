# Principle

RepoNerve has **one product release: v1.0.0**.

There is **no v1.x product roadmap**. Capabilities are delivered through **v0.x.0-alpha engineering iterations** that accumulate until all v1.0 acceptance criteria pass.

```text
v0.10.0-alpha  →  Foundation fixes
v0.11.0-alpha  →  Code Intelligence core
v0.12.0-alpha  →  Development Experience + linking
v0.13.0-alpha  →  Token Intelligence layer
v0.14.0-alpha  →  Evidence Graph + Session Memory
v0.15.0-alpha  →  Multi-language code intelligence
v1.0.0         →  Product release (all scope complete)
```

Alpha tags are **engineering checkpoints** for contributors. They are not partial product releases and do not satisfy v1.0 acceptance alone.

**Post-1.0:** Do not add new `v0.x-alpha` tags. Use semver per `docs/releases/versioning.md`.

# Complete v1.0 Scope

Everything below must ship before `v1.0.0` is tagged.

## Already Complete (v0.1.0-alpha – v1.0.0-alpha)

* Knowledge Preservation, Repository Intelligence
* Memory, Query, Context engines
* MCP server (43 tools)
* Ownership, Graph, Discovery, Reviewers, Change Planning
* Agent Context, Search, Session, Workflow (services)

## ISSUE-057 — Code Intelligence & Development Experience

* Code Intelligence (Go AST, symbols, call graph)
* Repository-Code Linking
* Feature Understanding
* Development Experience CLI (`ask`, `explain`, symbol explains, `plan`, `impact`, `review`)

## ISSUE-059 — Foundation Fixes (v0.10.0-alpha)

* Wire expertise detection into scan pipeline
* Expose agent search and graph impact via CLI
* Fix scan help text; wire or remove FTS5
* Unify impact authority (agent vs graph)

## ISSUE-060 — Token Intelligence (v0.13.0-alpha)

* Graph-aware context compression with token budgets
* `--format compact|prose|json` on CLI and MCP
* `reponerve hook install` (post-commit scan, session context inject)
* Incremental scan on commit
* RTK composition documentation

## ISSUE-061 — Evidence Graph & Session Memory (v0.14.0-alpha)

* Graph community detection and surprising connections
* Token-budget graph query (BFS/DFS `--budget N`)
* `reponerve explore` (CLI + HTML export)
* MCP: `discover_surprises`, `suggest_questions`, `query_graph`
* `reponerve remember` / `reponerve forget`
* Session writeback to repository memory
* Agent handoff bundles
* Fixed workflow templates (onboarding, review prep, change prep)

## ISSUE-062 — Multi-Language Code Intelligence (v0.15.0-alpha)

* Tree-sitter parsers: TypeScript, Python, Rust (minimum)
* Same authority boundaries as Go code intelligence
* Incremental indexing per language

# v1.0 Acceptance Gate

`v1.0.0` is tagged only when:

1. All issues above are complete
2. `docs/examples/development-experience.md` passes
3. Token economics targets documented in `docs/product/token-economics.md` are met
4. Architecture, feature, testing, performance, and documentation audits pass
5. `docs/releases/v1.0.0-checklist.md` is complete

# Agreed Out of v1.0

These are **not** in v1.0 scope. Confirmed product exclusions:

* Semantic / hybrid embedding search
* User-defined workflow composition
* Autonomous code modification or deployment
* Cloud-required core product
* Cross-repo enterprise federation

Also excluded: LLM-required parsing as primary authority.

Full detail: `docs/roadmap/v1.x-backlog.md`. Reconsideration requires post-v1.0 RFC.
