# Executive Summary

RepoNerve is a software understanding platform built around knowledge preservation.

RepoNerve preserves, organizes, and transfers software knowledge so that understanding survives beyond individual contributors.

RepoNerve serves as a **software memory system** — preserving understanding that would otherwise be lost through contributor turnover, architectural evolution, and documentation drift.

```text
Knowledge Preservation
        ↓
Software Memory
        ↓
Software Understanding
```

Repository Intelligence is complete. ISSUE-057 delivers the remaining v1.0 scope: Code Intelligence, Repository-Code Linking, Feature Understanding, and Development Experience.

Software Understanding is the primary product outcome. RepoNerve has one product release — v1.0.

---

# Vision

RepoNerve reduces the cost of software understanding.

Every developer and AI agent should understand what a repository does, how code works, why it exists, who owns it, what depends on it, and how to evolve it safely — without repeated repository exploration.

# Knowledge Preservation

Software knowledge must survive team changes, contributor turnover, architectural evolution, and long-lived repositories.

RepoNerve acts as a software memory system — preserving repository knowledge and code structure so understanding remains available when original authors depart.

---

# Knowledge Transfer

RepoNerve helps new developers onboard, reviewers gain context, architects assess impact, managers understand ownership, and AI agents obtain development context — without requiring access to original contributors.

Development Experience is the primary transfer mechanism.

# AI Agent Context

RepoNerve reduces repository exploration for AI systems. The objective is understanding — not retrieval alone.

Agents should obtain code, repository, ownership, architectural, and change context before implementation work.

---

# Differentiation

Most tools: Code Graph → Retrieval → LLM Context

RepoNerve: Knowledge Preservation → Software Memory → Code + Repository + Ownership + Architectural + Change Intelligence → Software Understanding

RepoNerve answers *why*, *who*, and *what breaks* — not only *where* — with mandatory evidence.

See `docs/product/market-positioning.md`.

# Token Economics

RepoNerve minimizes token consumption for software understanding:

* Deterministic `scan` (no LLM required for indexing)
* Bounded MCP context packs
* Persistent repository memory across agent sessions

Premium models stay within limits when understanding is delivered before implementation begins.

See `docs/product/token-economics.md`.
