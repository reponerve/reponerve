# RepoNerve Out-of-Scope Items

Version: 1.2

Status: Current

Updated: 2026-06-25

---

# Purpose

RepoNerve **v1.0.0 shipped** 2026-06-18. Post-1.0 capabilities ship via **semver** (`v1.0.x`, `v1.1.0`, …) with RFC approval — see `docs/releases/versioning.md`.

This document lists capabilities **not planned** for the current release line. Reconsideration requires a new RFC.

**Shipped post-1.0 (not listed here):** bounded DE, feature intelligence v2, native discipline, reuse/ship checks, doctor, scoped scan, npm distribution, local explore UI — see `docs/product/implementation-status.md`.

---

# Out of Scope

## Semantic Search

Embedding-based or vector retrieval as a primary authority.

## Hybrid Search

Combined deterministic and semantic retrieval adapters where vectors override evidence.

## User Defined Workflows

Custom workflow composition beyond the fixed workflow template set.

## Autonomous Code Modification

RepoNerve does not autonomously write, commit, or deploy code.

## Cloud-Required Core Product

Core product must run local-first without mandatory SaaS.

## Cross-Repo Enterprise Federation

Org-wide multi-repository federation at Sourcegraph scale.

## Full-Product Graph Explorer

Enterprise-grade graph product beyond the local `reponerve explore` UI (v1.5.0 ships a capped local browser; full product scope remains out of scope).

# Rule

Post-1.0 delivery uses **semver** — not new `v0.x-alpha` tags.

New capabilities require an RFC (see `docs/governance/rfc-process.md`) and a row in `docs/releases/versioning.md` before tagging.

Historical v1.0 iteration context: `docs/roadmap/v1.0-iteration-plan.md`.
